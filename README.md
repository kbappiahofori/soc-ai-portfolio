# SOC-AI Portfolio

## Overview
This repository is a public-facing portfolio of my hands-on projects in SOC and cybersecurity.  
It showcases technical labs and simulations with an emphasis on network monitoring,
investigation workflows, and SOC-style incident documentation. Where applicable,
AI-assisted workflows are used to improve clarity, consistency, and professional
presentation of security reporting.

## Structure
Projects are organized into clearly defined folders:

- [01-network-traffic-monitoring](01-network-traffic-monitoring)  
  Cisco Packet Tracer labs focused on traffic monitoring and analysis, including VLANs,
  ACLs, configurations, screenshots, and SOC-style documentation.

- [02-ai-assisted-soc-reporting](02-ai-assisted-soc-reporting)  
  SOC incident and alert reports emphasizing documentation quality, structured reporting,
  and comparison between manual and assisted workflows.

- [03-soc-investigation-labs](03-soc-investigation-labs)  
  TryHackMe and BTLO write-ups documenting investigative reasoning, log interpretation,
  and escalation considerations from a SOC perspective.

- [04-web-security-observations](04-web-security-observations)  
  Web security exercises (e.g., OWASP Juice Shop) framed from a defensive SOC viewpoint,
  focusing on how common attack patterns may surface in alerts and logs.

- [templates](templates)  
  Reusable documentation templates for SOC incident reports, lab write-ups,
  and Packet Tracer simulations.

## Goals
🎯 Build a public-facing portfolio that demonstrates SOC-relevant technical skills,
clear and disciplined documentation, and a foundation for long-term growth in
information security.  
This repository is designed to scale over time as additional labs, simulations,
and reports are completed.
